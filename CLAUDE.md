# Web Front-end Training

## プロジェクト概要

エンジニア初心者が Next.js を使ったプロジェクトに参加するための、Web フロントエンド学習コンテンツ。
VitePress で構築されたドキュメントサイトとして提供する。

## ターゲット

- エンジニア初心者（プログラミング経験が浅い、またはバックエンド中心だった人）
- これから Next.js を使うプロジェクトに配属される予定の人
- HTML/CSS/JavaScript の基礎知識が不十分な人

## コンテンツの使われ方

- 1日1レッスン、約15分で取り組む想定
- 毎日少しずつ進めてもらう（一気にやるものではない）
- 順番に進める前提（前の Day の知識を前提にしてよい）
- 最終的に Next.js のプロジェクトで自走できるレベルを目指す

## 執筆方針

- **仕組みの理解を重視**: AI ツールで動くコードは誰でも作れる時代。「動いた」で終わらせず、なぜそう書くのか・どういう仕組みで動いているのかを理解させることがこのコンテンツの価値。コードの書き方だけでなく、ブラウザやフレームワークが裏で何をしているかを伝える
- **初心者目線**: 専門用語は初出時に必ず平易な言葉で説明する。「知っていて当然」という態度を取らない
- **動くコード**: コード例は断片ではなく、コピーすればそのまま動く完全なコードを載せる。読みながらマネして動かせる内容にする
- **演習は不要**: 独立した演習セクションは設けない。説明の流れの中で自然に手を動かせるようにする
- **簡潔に**: 15分で読み切れるボリュームを意識する。網羅性より「今日はこれだけ覚えれば OK」を重視する
- **繋がりを意識**: 前日のレッスンで学んだ内容を活かす場面があれば積極的に触れる
- **アクセシビリティを意識**: コード例ではセマンティックな HTML を使い、適切な alt 属性、aria 属性、キーボード操作への配慮などを自然に盛り込む。「なぜそう書くのか」をアクセシビリティの観点から説明する。専用レッスンとしてではなく、各レッスンの中で当たり前の習慣として扱う
