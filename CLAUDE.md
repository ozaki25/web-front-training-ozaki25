# Web Front-end Training

## プロジェクト概要

エンジニア初心者が Next.js を使ったプロジェクトに参加するための、Web フロントエンド学習コンテンツ。
VitePress で構築されたドキュメントサイトとして提供する。

## ターゲット

- エンジニア初心者。AI 丸投げ開発の経験が多少ある程度
- これから Next.js（App Router）を使うプロジェクトに配属される予定の人
- HTML/CSS/JavaScript の基礎を自分の言葉で説明できないレベル

## 配属先の技術スタック

- Next.js（App Router、最新バージョンに追従）
- TypeScript
- Tailwind CSS

## ゴール

フレームワーク・ライブラリ・Web 標準を正しく理解し、アーキテクチャの議論に参加できるレベル。具体的には:

- Server Components と Client Components の使い分けを説明・議論できる
- データ取得の設計（どこで fetch するか、キャッシュ戦略）について話せる
- ディレクトリ構成やコンポーネント分割の意図を理解できる
- パフォーマンス、アクセシビリティ、テスト戦略、CI/CD などの技術選定・設計の議論に参加できる
- 「なぜその技術を使うのか」の背景を理解している

## コンテンツの使われ方

- 1日1レッスン、約15分で取り組む想定
- 毎日少しずつ進めてもらう（一気にやるものではない）
- 順番に進める前提（前の Day の知識を前提にしてよい）
- 平日のみ、約2ヶ月（約40日分）で完了する想定

## 執筆方針

- **常に最新版を参照する**: コンテンツで扱うライブラリ・フレームワーク・Web 標準は、必ず執筆時点の最新安定版に基づいて書く。古いバージョンの情報を載せない。レッスン作成時には公式ドキュメントまたは Web 検索で最新の API・仕様を確認すること
- **仕組みの理解を重視**: AI ツールで動くコードは誰でも作れる時代。「動いた」で終わらせず、なぜそう書くのか・どういう仕組みで動いているのかを理解させることがこのコンテンツの価値。コードの書き方だけでなく、ブラウザやフレームワークが裏で何をしているかを伝える
- **素の技術はフレームワーク理解の前提条件**: 素の HTML/CSS/JavaScript の仕組みを知らなければ、React や Next.js が何を解決しているか理解できない。CSS のグローバルスコープ問題を知らなければ CSS Modules や Tailwind の価値がわからず、DOM 操作の煩雑さを知らなければ宣言的 UI のありがたみがわからない。基礎は「教養」ではなく、フレームワークを正しく理解するための必須の土台として位置づける
- **初心者目線**: 専門用語は初出時に必ず平易な言葉で説明する。「知っていて当然」という態度を取らない
- **動くコード**: コード例は断片ではなく、コピーすればそのまま動く完全なコードを載せる。読みながらマネして動かせる内容にする
- **演習は不要**: 独立した演習セクションは設けない。説明の流れの中で自然に手を動かせるようにする
- **簡潔に**: 15分で読み切れるボリュームを意識する。網羅性より「今日はこれだけ覚えれば OK」を重視する
- **繋がりを意識**: 前日のレッスンで学んだ内容を活かす場面があれば積極的に触れる
- **アクセシビリティを意識**: コード例ではセマンティックな HTML を使い、適切な alt 属性、aria 属性、キーボード操作への配慮などを自然に盛り込む。「なぜそう書くのか」をアクセシビリティの観点から説明する。専用レッスンとしてではなく、各レッスンの中で当たり前の習慣として扱う
