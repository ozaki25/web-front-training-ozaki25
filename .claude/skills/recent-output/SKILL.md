# 直近出力の再表示スキル

Claude Code（web 版）は、非常に長いセッションだと画面に直近の履歴を描画しきれないことがある。
このスキルは、ディスクに保存された会話トランスクリプト（`.jsonl`）から直近の assistant 出力を
**一字一句そのまま**取り出して、いま見えているメッセージに貼り直す。

**重要**: 記憶から書き直してはいけない（文脈は要約で圧縮されており不正確になる）。必ず下の
スクリプトで実データを読むこと。

## 手順

1. まず索引を出す（引数なし＝直近 20 件の先頭だけ）:

   ```bash
   node .claude/hooks/recent-output.js
   ```

   出力は `[番号] 時刻 (文字数) 冒頭60字` の一覧。これをそのままユーザーに提示する。

2. ユーザーが見たい出力を指定したら、全文を取り出して**そのまま**貼る:

   - 冒頭の文言で指定された場合（推奨・安定）:

     ```bash
     node .claude/hooks/recent-output.js --match "テスト系だけ、推奨読了順"
     ```

   - 「最新」なら:

     ```bash
     node .claude/hooks/recent-output.js --last
     ```

   - 索引の番号で指定された場合（同じターン内なら有効。新しい発言が増えると番号はずれる）:

     ```bash
     node .claude/hooks/recent-output.js --full 3
     ```

   - 件数を増やして索引を見たいとき: `--n 40`

3. 取り出した本文は、要約・整形せず**逐語でそのまま**提示する。

## 注意

- **番号は安定しない**: 新しいやり取りが増えると `--full` の番号はずれる。全文指定は
  なるべく `--match "冒頭の文言"` を使う（こちらは内容一致なので安定）。
- スクリプトは `/root/.claude/projects/*/*.jsonl` の中で最終更新が最新のファイル（＝現在の
  セッション）を自動選択する。セッション ID をハードコードしない。
- これは描画不具合の回避策。根本的には新しいセッションを開けば直近履歴は普通に見える
  （作業成果は git に残るので移行しても失われない）。それも併せて案内してよい。
