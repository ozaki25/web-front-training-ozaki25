# Day 10: テーブル — データの構造を伝えるマークアップ

## 今日のゴール

- テーブルの各タグの役割と正しい構造を知る
- `scope` 属性や `caption` がアクセシビリティに与える効果を知る
- テーブルをレイアウト目的で使ってはいけない理由を知る

## テーブルは「データの関係性」を表現するもの

Day 3 でテーブルの基本構造を学びました。今日はもう少し踏み込んで、テーブルが**データの行と列の関係性をマシンリーダブル（機械が読める形）で表現する仕組み**であることを見ていきます。

たとえば、こんな表を HTML で書くとします。

| 社員名 | 部署 | 入社年 |
|--------|------|--------|
| 山田太郎 | 開発部 | 2023 |
| 佐藤花子 | デザイン部 | 2024 |

人間の目にはすぐに「山田太郎は開発部で2023年入社」とわかりますが、スクリーンリーダーやプログラムがこの関係性を正しく読み取るためには、HTML の構造で明示する必要があります。

## テーブルの基本構造

```html
<table>
  <thead>
    <tr>
      <th>社員名</th>
      <th>部署</th>
      <th>入社年</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>山田太郎</td>
      <td>開発部</td>
      <td>2023</td>
    </tr>
    <tr>
      <td>佐藤花子</td>
      <td>デザイン部</td>
      <td>2024</td>
    </tr>
  </tbody>
</table>
```

### 各タグの役割

| タグ | 意味 | 役割 |
|------|------|------|
| `<table>` | テーブル全体 | テーブルのルート要素 |
| `<thead>` | ヘッダー部分 | 見出し行のグループ。ブラウザが印刷時にページごとにヘッダーを繰り返す手がかりにもなる |
| `<tbody>` | データ部分 | データ行のグループ。省略してもブラウザが自動補完する（Day 9 で学んだ DOM の自動補完） |
| `<tfoot>` | フッター部分 | 合計行などのグループ |
| `<tr>` | 行（table row） | テーブルの1行 |
| `<th>` | 見出しセル（table header） | その列（または行）が何のデータかを表す |
| `<td>` | データセル（table data） | 実際のデータ |

## th と td の違い — 見出しとデータ

`<th>` と `<td>` は見た目は似ていますが、役割が全く異なります。

```html
<tr>
  <th>社員名</th>    <!-- これは「見出し」 -->
  <td>山田太郎</td>  <!-- これは「データ」 -->
</tr>
```

- `<th>` はデフォルトで**太字・中央揃え**になりますが、大事なのは見た目ではなく**意味**です
- スクリーンリーダーは `<th>` を「見出しセル」として認識し、データセルを読み上げるときに対応する見出しを添えて読み上げます

たとえば、「開発部」というセルにフォーカスが当たったとき:

- `<th>` が正しく使われていれば:「部署、開発部」と読み上げる
- すべてが `<td>` だと:「開発部」としか読み上げず、何の列の値かわからない

## scope 属性 — 見出しが行と列のどちらに対応するかを明示する

`<th>` に `scope` 属性を付けると、その見出しが**列の見出し**なのか**行の見出し**なのかを明示できます。

### 列の見出し（scope="col"）

```html
<table>
  <thead>
    <tr>
      <th scope="col">社員名</th>
      <th scope="col">部署</th>
      <th scope="col">入社年</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>山田太郎</td>
      <td>開発部</td>
      <td>2023</td>
    </tr>
  </tbody>
</table>
```

`scope="col"` は「この見出しは列（column）全体に対応する」という意味です。

### 行の見出し（scope="row"）

行の先頭にも見出しがある場合、`scope="row"` を使います。

```html
<table>
  <thead>
    <tr>
      <th scope="col">商品</th>
      <th scope="col">1月</th>
      <th scope="col">2月</th>
      <th scope="col">3月</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">りんご</th>
      <td>100個</td>
      <td>150個</td>
      <td>120個</td>
    </tr>
    <tr>
      <th scope="row">みかん</th>
      <td>200個</td>
      <td>180個</td>
      <td>210個</td>
    </tr>
  </tbody>
</table>
```

この構造なら、「150個」というセルにフォーカスが当たったとき、スクリーンリーダーは「りんご、2月、150個」と読み上げることができます。行と列の両方の見出しが紐付いているからです。

## caption — テーブルのタイトル

`<caption>` はテーブル全体の説明を付けるタグです。

```html
<table>
  <caption>2026年 第1四半期の果物販売数</caption>
  <thead>
    <tr>
      <th scope="col">商品</th>
      <th scope="col">1月</th>
      <th scope="col">2月</th>
      <th scope="col">3月</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">りんご</th>
      <td>100個</td>
      <td>150個</td>
      <td>120個</td>
    </tr>
    <tr>
      <th scope="row">みかん</th>
      <td>200個</td>
      <td>180個</td>
      <td>210個</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <th scope="row">合計</th>
      <td>300個</td>
      <td>330個</td>
      <td>330個</td>
    </tr>
  </tfoot>
</table>
```

`<caption>` は `<table>` の**直後**（`<thead>` より前）に書きます。スクリーンリーダーはテーブルに到達したとき、まず `<caption>` の内容を読み上げるので、これからどんなデータが続くかをユーザーに伝えられます。

この例では `<tfoot>` も使っています。合計行のように「データのまとめ」にあたる行を `<tfoot>` で囲むと、テーブルの構造がより明確になります。

## テーブルをレイアウトに使ってはいけない理由

1990年代から2000年代初頭、CSS のレイアウト機能が貧弱だった時代、テーブルは**ページ全体のレイアウト**に使われていました。

```html
<!-- ❌ レイアウト目的のテーブル（やってはいけない） -->
<table>
  <tr>
    <td>サイドバー</td>
    <td>メインコンテンツ</td>
    <td>広告</td>
  </tr>
</table>
```

なぜこれがダメなのか、3 つの理由があります。

### 1. アクセシビリティの問題

スクリーンリーダーはテーブルを「表形式のデータ」として解釈します。レイアウト用テーブルだと、ナビゲーションやメインコンテンツが「データセル」として読み上げられ、ページの構造がまったく伝わりません。

### 2. セマンティクスの崩壊

HTML の要素にはそれぞれ意味があります（セマンティクス）。`<table>` は「表形式のデータ」を意味するので、レイアウトに使うと**意味と見た目が乖離**します。検索エンジンもページの構造を正しく理解できなくなります。

### 3. レスポンシブ対応の困難さ

テーブルレイアウトはスマートフォンの画面幅に柔軟に対応できません。Day 6 で学んだ Flexbox や Day 7 の CSS Grid を使えば、同じレイアウトをよりシンプルに、かつレスポンシブに実現できます。

```html
<!-- ✅ CSS Grid でレイアウト -->
<div style="display: grid; grid-template-columns: 200px 1fr 200px; gap: 16px;">
  <aside>サイドバー</aside>
  <main>メインコンテンツ</main>
  <aside>広告</aside>
</div>
```

**テーブルはデータのためのもの。レイアウトは CSS の仕事。** これは現代の Web 開発の基本ルールです。

## まとめ

- テーブルはレイアウトツールではなく、データの行と列の関係性を表現するもの
- `<th>` は見出しセル、`<td>` はデータセル。スクリーンリーダーがデータと見出しを紐付けて読み上げるために重要
- `scope` 属性（`col` / `row`）で見出しが列・行のどちらに対応するかを明示する
- `<caption>` でテーブルの説明を付けると、スクリーンリーダーが最初に読み上げてくれる
- テーブルのレイアウト利用はアクセシビリティ・セマンティクス・レスポンシブの3点でダメ。レイアウトには Flexbox や CSS Grid を使う
