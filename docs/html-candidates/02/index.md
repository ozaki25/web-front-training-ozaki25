# Day X: 見出しレベルの設計 — h1〜h6 は文字サイズではなくページの目次

## 今日のゴール

- h1〜h6 が「文字の大きさ」ではなく「ページの階層構造」を表すことを知る
- 見出しレベルを飛ばしてはいけない理由を知る
- React のコンポーネント分割で見出しレベルが壊れやすい問題を知る

## h1〜h6 は「階層構造」を表すタグ

HTML には `<h1>` から `<h6>` まで、6 段階の見出しタグがあります。数字が小さいほど上位の見出しです。

```html
<h1>Web フロントエンド入門</h1>
  <h2>HTML の基礎</h2>
    <h3>見出しタグ</h3>
    <h3>段落タグ</h3>
  <h2>CSS の基礎</h2>
    <h3>セレクタ</h3>
    <h3>ボックスモデル</h3>
```

このコードを見ると、見出しがページの **目次（アウトライン）** を構成していることがわかります。本の目次と同じで、「章 → 節 → 項」のように階層が深くなっていきます。

上の例をアウトラインとして見ると、こうなります。

```
Web フロントエンド入門
├── HTML の基礎
│   ├── 見出しタグ
│   └── 段落タグ
└── CSS の基礎
    ├── セレクタ
    └── ボックスモデル
```

**h1〜h6 はこの階層を表すためのタグ**です。文字の大きさを変えるためのタグではありません。

## 「文字サイズのタグ」と誤解されやすい理由

ブラウザは見出しタグにデフォルトのスタイルを付けます。`<h1>` は大きく太い文字、`<h2>` はそれよりやや小さい文字 ── というように、数字が大きくなるほど文字が小さくなります。

この見た目のせいで、「文字を大きくしたいから `<h1>` を使おう」「小さい見出しにしたいから `<h4>` にしよう」という使い方をしてしまうことがあります。

```html
<!-- ❌ 文字サイズで見出しレベルを選んでいる -->
<h1>サイト名</h1>
<h4>ここは小さめに表示したい見出し</h4>
<p>本文...</p>
<h2>ここは大きめに表示したい見出し</h2>
<p>本文...</p>
```

このコードでは、h1 → h4 → h2 と見出しレベルが飛び飛びになっています。見た目は CSS で思い通りにできますが、ページの階層構造はめちゃくちゃです。

**文字の大きさを変えたいだけなら CSS を使います。**

```css
/* 見出しの見た目は CSS で自由に変えられる */
h2 {
  font-size: 18px;  /* h2 でも小さい文字にできる */
}

h3 {
  font-size: 24px;  /* h3 でも大きい文字にできる */
}
```

タグは「意味」を、CSS は「見た目」を担当する ── これは Day 1 で学んだ HTML と CSS の役割分担そのものです。

## 見出しレベルを飛ばしてはいけない理由

見出しレベルは **h1 → h2 → h3** のように順番に深くする必要があります。h1 の次にいきなり h3 を使ったり、h2 を飛ばして h4 を使ったりしてはいけません。

```html
<!-- ❌ h2 を飛ばしている -->
<h1>お知らせ</h1>
<h3>2026年4月のお知らせ</h3>
<p>新機能をリリースしました。</p>

<!-- ✅ 正しい階層 -->
<h1>お知らせ</h1>
<h2>2026年4月のお知らせ</h2>
<p>新機能をリリースしました。</p>
```

なぜ飛ばしてはいけないのでしょうか。

### 1. スクリーンリーダーが見出しでナビゲーションする

スクリーンリーダーには **見出し一覧を表示してページ内を移動する機能** があります。ユーザーは「h2 の一覧を表示」「次の h3 に移動」といった操作でページの構造を把握し、目的の場所にジャンプします。

見出しレベルが飛んでいると、「ここに h2 がないのは情報が欠けているのか、それとも単に書き方の問題なのか」が判断できません。

### 2. ページのアウトラインが壊れる

見出しレベルが飛ぶと、ページの論理的な階層構造が壊れます。

```
❌ 飛ばした場合のアウトライン:
お知らせ（h1）
    ???（h2 がない）
        2026年4月のお知らせ（h3）

✅ 正しいアウトライン:
お知らせ（h1）
└── 2026年4月のお知らせ（h2）
```

### 3. 深い階層から浅い階層に戻るのは OK

注意が必要なのは「飛ばす」ことです。深い階層から浅い階層に戻るのは問題ありません。

```html
<!-- ✅ h3 から h2 に戻るのは正しい -->
<h1>料理レシピ</h1>

<h2>和食</h2>
<h3>肉じゃが</h3>
<p>材料：じゃがいも、牛肉、にんじん...</p>
<h3>味噌汁</h3>
<p>材料：豆腐、わかめ、味噌...</p>

<h2>洋食</h2>
<h3>ハンバーグ</h3>
<p>材料：ひき肉、玉ねぎ、パン粉...</p>
```

「和食」の節が終わって「洋食」の節に入るとき、h3 → h2 に戻っています。これは本の目次でも自然な構造です。

## h1 はページに1つが基本

`<h1>` はそのページで最も重要な見出し、つまり **ページのタイトル** です。

```html
<!-- ✅ h1 はページのタイトルとして1つだけ -->
<body>
  <header>
    <nav><!-- サイトのナビゲーション --></nav>
  </header>
  <main>
    <h1>会社概要</h1>
    <h2>事業内容</h2>
    <p>...</p>
    <h2>アクセス</h2>
    <p>...</p>
  </main>
</body>
```

HTML の仕様上、h1 を複数使うこと自体はエラーになりません。しかし、**ページに h1 が 1 つ** というのは広く受け入れられた慣習です。検索エンジンもページの h1 を重要な手がかりとして扱います。

よくある間違いとして、サイト名とページタイトルの両方を h1 にしてしまうケースがあります。

```html
<!-- ❌ h1 が 2 つある -->
<header>
  <h1>My Site</h1>
</header>
<main>
  <h1>会社概要</h1>
</main>

<!-- ✅ サイト名はリンク付きのテキスト、h1 はページタイトル -->
<header>
  <p><a href="/">My Site</a></p>
</header>
<main>
  <h1>会社概要</h1>
</main>
```

トップページではサイト名を h1 にするのが自然ですが、それ以外のページでは、**そのページ固有のタイトルを h1 にする**のが一般的です。

## React のコンポーネント分割で見出しレベルが壊れやすい

React ではページを小さなコンポーネントに分割します。ここで、見出しレベルの問題が起きやすくなります。

```tsx
// SectionTitle コンポーネント — 常に h2 を出力する
function SectionTitle({ children }: { children: React.ReactNode }) {
  return <h2>{children}</h2>;
}
```

このコンポーネントは常に `<h2>` を出力します。ところが、使う場所によっては h3 が正しいこともあります。

```tsx
function Page() {
  return (
    <main>
      <h1>料理レシピ</h1>

      {/* ここでは h2 が正しい */}
      <SectionTitle>和食</SectionTitle>

      {/* ここでは本当は h3 が欲しいが、h2 が出力されてしまう */}
      <section>
        <SectionTitle>肉じゃが</SectionTitle>
        <p>材料：じゃがいも、牛肉、にんじん...</p>
      </section>
    </main>
  );
}
```

出力される HTML は h1 → h2 → h2 となり、「肉じゃが」が「和食」の下位ではなく同じレベルに見えてしまいます。

### 対策: 見出しレベルを props で受け取る

```tsx
function SectionTitle({
  level,
  children,
}: {
  level: 1 | 2 | 3 | 4 | 5 | 6;
  children: React.ReactNode;
}) {
  const Tag = `h${level}` as const;
  return <Tag>{children}</Tag>;
}

function Page() {
  return (
    <main>
      <h1>料理レシピ</h1>
      <SectionTitle level={2}>和食</SectionTitle>
      <section>
        <SectionTitle level={3}>肉じゃが</SectionTitle>
        <p>材料：じゃがいも、牛肉、にんじん...</p>
      </section>
    </main>
  );
}
```

見出しレベルを props で指定できるようにすれば、使う場所に応じて正しい階層を保てます。コンポーネントを作るときは「このコンポーネントがページのどの階層で使われるか」を意識することが大切です。

## 見出し構造のチェック方法

ページの見出し構造が正しいかどうかは、ブラウザの開発者ツールや拡張機能で確認できます。

- **Chrome DevTools**: Elements パネルで `h1`〜`h6` を検索する
- **HeadingsMap**（ブラウザ拡張機能）: ページの見出しをツリー形式で一覧表示してくれる
- **axe DevTools**（ブラウザ拡張機能）: 見出しレベルの飛ばしを自動で検出してくれる

AI にコードを生成させた後も、見出し構造を確認する習慣を持つと、アクセシビリティの問題を早期に発見できます。

## まとめ

- h1〜h6 はページの階層構造（アウトライン）を表すタグであり、文字サイズのタグではない
- 見出しレベルは飛ばさない（h1 → h3 のように飛ばすのは NG、h3 → h2 のように戻るのは OK）
- h1 はページに 1 つが基本。そのページ固有のタイトルを h1 にする
- 文字の大きさを変えたいなら CSS の `font-size` を使う
- React のコンポーネント分割では見出しレベルが壊れやすい。props でレベルを指定できる設計にすると安全
- スクリーンリーダーのユーザーは見出し一覧でページ内を移動するため、正しい見出し構造はアクセシビリティの基本
