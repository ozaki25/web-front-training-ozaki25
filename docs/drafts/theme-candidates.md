# テーマ候補リスト v3（広域プール）

全量データ（蔵書約130冊・イベントメモ553本）と本人インタビューに基づく候補プール。
方針は「**まずは広く、その後選別**」（本人確認済み）。v3 は「広く」の成果物であり、約100テーマを除外せずに保持する。選別は次フェーズで行う。

> ステータス: 候補段階。内容・順序・数はいつでも変わりうる。
> プロセスの記録は `theme-sprint-log.md` を参照。

## 選定軸

1. **仕組みから語れるか**: 「〜って何なの？」「〜ってどうなってるの？」と疑問形で掘れるテーマか。手順の説明しかできないテーマは弱い
2. **ペルソナが共感できるか**: AI 丸投げ経験者が「見たことある」「困ったことある」から入れるか
3. **作者の知見が深いか**: 発信履歴（Qiita 111記事、handson 10本超）と蔵書・イベント参加の蓄積がある領域か

優先度仮説の意味: **高** = 三軸が揃う / **中** = 二軸 / **低** = 一軸以下だが再構成の余地あり / **境界** = 別教材スコープと隣接。いずれも除外ではない。

### 教材の住み分け（重要）

作者は教材を複数リポジトリに分けている。本教材のスコープ判断に直結する。

| リポジトリ | 役割 |
|-----------|------|
| web-front-training（本教材） | 読むだけで完結する知識の引き出し |
| web-front-handson | 手を動かすハンズオン（姉妹教材） |
| color-coordination-training / ux-certification-basics | 配色・UX・デザイン領域 |

→ **デザイン・UX 系のテーマは別教材の領域**なので本教材では扱わない。アクセシビリティは「フロントエンド実装の一部」として本教材のスコープ（既に全レッスン方針に組み込み済み）。

### インプットとアウトプットの非対称

蔵書はアジャイル・組織・マネジメントまで幅広いが、**発信・教材化は一貫して「仕組みを噛み砕く」フロントエンド系のみ**。プロセス系・組織系は読書で学ぶ領域であり教材化していない。本教材でも同じ線引きを優先度仮説に反映する（プロセス系 = 低）。

## 公開順序の考え方

厳密な順序は publish 時に決める。大きな流れ:

1. **JS 基礎の補完**(下書き 005, 006 → A1, A2)
2. **React の核**(A3 → A4 → A5 → 下書き 015, 016 → A6, A7)
3. **仕組みの深掘りを混ぜる**(S1, S3, B1, B5, B6, N05 を React 基礎の合間に)
4. **Next.js 実践**(下書き 013 → A8〜A12 → S4)
5. **広がり**(S5〜S11, B2〜B4, B7, N 系を読者の遭遇順に)

単調にならないよう、React 基礎の連続の合間に歴史もの・仕組みものを挟んで緩急をつける。
