# UI デザインの法則 — 使いやすさは人間側の仕組みで決まる

## 今日のゴール

- UI の配置に「理由のある定石」があることを知る
- 代表的な 4 つの法則を知り、見慣れた画面で発見できるようになる
- AI に UI を頼むとき「法則の名前」で指示できるようになる

## 「なんとなく使いやすい」を分解する

使いやすいアプリと使いにくいアプリの違いは、しばしば「なんとなく」としか言語化できません。しかし UI デザインの世界では、この「なんとなく」の多くに**名前と法則**が付いています。法則の源は流行や好みではなく、**人間の体と認知の仕組み**（指の動かし方、短期記憶の容量、過去の経験からの予測）です。ブラウザの仕組みを知るとコードが読めるように、人間側の仕組みを知ると UI が読めるようになります。

法則を知ると 2 つ良いことがあります。「なんとなく使いにくい」を**言葉で指摘できる**ようになること。そして AI に「なんとなく良い感じに」ではなく**法則の名前で指示できる**ようになることです。

## 法則 1: フィッツの法則 — 大きく、近く

> **ターゲット（押したいもの）は、大きいほど、近いほど、速く押せる。**

当たり前に聞こえますが、この法則の帰結は意外なほど設計に響きます。

- フォームの「送信」ボタンは、最後の入力欄の**すぐ近く**に置く（離れていると目線と手が往復する）
- モバイルのタップ領域は、最低 **44 × 44 px** は確保する（Apple の HIG の推奨値）。テキストリンクが小さすぎると、隣のリンクを誤タップする
- 「削除」のような破壊的操作は、よく使うボタンから**距離を取る**ことで誤操作を防げる

ボタン群の間隔が狭いとき、「フィッツの法則的にタップ領域が足りない」と言えると、感覚ではなく根拠のある指摘になります。

## 法則 2: ヤコブの法則 — ユーザーは他のサイトで学んでいる

> **ユーザーは、あなたのサイト以外のサイトでほとんどの時間を過ごしている。だから他と同じように動くことを期待する。**

- ロゴを押すとトップページに戻る
- カートのアイコンは右上にある
- 検索は虫眼鏡のアイコン
- リンクは下線付きの青文字

これらは「正解」ではなく「**期待**」です。独創的な配置にすると、ユーザーは過去の経験が使えず、毎回「これはどこ？」と探すコストを払います。

AI に UI を作らせると、たまに独創的な配置をしてきます。「ヤコブの法則に沿って、一般的な配置にして」と頼めば、ユーザーの学習コストを下げるデザインに寄ります。

## 法則 3: ミラーの法則 — 一度に覚えられるのは 7±2

> **人間が短期記憶に保持できる情報の塊は、おおむね 7 個前後。**

- ナビゲーションの項目が 15 個並んでいると、全体を把握できない
- フォームの入力項目が 20 個一列に続くと、「今どこまで進んだか」が分からなくなる

対策は**グループ化**です。15 項目のナビを 3〜4 カテゴリに分ける、20 項目のフォームをステップに分ける。人間の短期記憶の限界に合わせて情報を分割します。

設定画面に 30 個のトグルが並んでいたら、「カテゴリで分けて」と言うだけで使いやすさが大きく変わります。

## 法則 4: 近接の法則 — 近いものは仲間に見える

> **物理的に近い要素は、関連のあるグループだと知覚される。**

```
ユーザー名: [        ]   ← この 2 つは仲間に見える
メール:     [        ]

                          ← 余白があると区切りに見える
パスワード: [        ]
確認:       [        ]
```

ラベルと入力欄が近ければ「このラベルはこの欄のもの」と分かります。逆にラベルと入力欄の間が広く、入力欄同士が近いと、「どのラベルがどの欄？」と迷います。

**余白は飾りではなく、グループ構造を伝える道具**です。AI に「余白を調整して」ではなく「**関連する要素を近づけて、グループ間に余白を入れて**」と頼むと、近接の法則に沿った配置になります。

## 法則は「根拠」になる

4 つの法則に共通するのは、**「感覚の話」を「根拠のある話」に変える**力です。

| 感覚 | 法則で言い換え |
|------|-------------|
| 「このボタン、押しにくい」 | フィッツの法則: ターゲットが小さすぎる、または遠い |
| 「なんか使いにくい」 | ヤコブの法則: 一般的な配置と違う |
| 「情報が多すぎて迷う」 | ミラーの法則: 一度に見せる項目数が多すぎる |
| 「どれとどれが関係あるか分からない」 | 近接の法則: グループの余白が足りない |

チームの会話で「使いにくい」だけだと主観の議論になりがちですが、法則の名前を出せると**共通の物差し**で話せます。

## まとめ

- フィッツの法則: 押したいものは大きく、近く。タップ領域 44px 以上
- ヤコブの法則: 他のサイトと同じ配置をユーザーは期待する
- ミラーの法則: 一度に 7±2 個。多いならグループ化
- 近接の法則: 近い = 仲間。余白はグループ構造を伝える道具
