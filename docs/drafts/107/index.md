# Router Cache — ブラウザが画面遷移を速くする仕組み

## 今日のゴール

- Router Cache が「訪れたページの表示をブラウザ側に控えておく仕組み」だと知る
- 控えがないとき（毎回取得）とあるとき（瞬時に戻れる）の違いを知る
- 古い控えを捨てる方法（再検証・`router.refresh`）を知る

## 「戻る」が一瞬で表示される理由

App Router のサイトで、一覧ページから詳細ページに進み、ブラウザの「戻る」で一覧に戻ると、**読み込みなしで一瞬で表示される**。リンクをホバーしただけで遷移先が先に用意されていることもあります。こうした「速い遷移」を支えているのが **Router Cache** です。

これまで見てきたデータや HTML の保存はサーバー側の話でしたが、Router Cache は**ブラウザ側**にあります。一度訪れたページの表示を、ブラウザがメモリに控えておく仕組みです。

## 控えがないと、毎回サーバーに取りに行く

もし控えがなければ、画面遷移のたびにサーバーへリクエストを送り、ページの中身を取り直すことになります。

- 戻る・進むのたびにサーバーとの往復が発生する
- ネットワークの遅延がそのまま待ち時間になる
- タブの切り替えのような操作でも、毎回ロードが挟まる

一覧 → 詳細 → 戻る、を繰り返すような操作では、この往復が積み重なって体感が重くなります。

## 控えがあると、瞬時に遷移できる

Router Cache は、訪れたページの表示をブラウザが**一時的に保持**します。同じページに戻るときは、サーバーに行かずに控えを表示します。

```mermaid

  A["一覧を表示<br>（控えを保持）"] --> B["詳細へ進む<br>（控えを保持）"]
  B -->|"戻る"| C["一覧の控えを表示<br>サーバーに行かない＝一瞬"]
  style A fill:#dcfce7,color:#1e293b,stroke:#22c55e
  style C fill:#dcfce7,color:#1e293b,stroke:#22c55e
```

これにより、戻る・進むやネスト内の切り替えが瞬時になります。ブラウザが控えを持つので、サーバーやネットワークを介さずに表示できます。

## 控えも古くなる

便利な反面、控えはあくまで「前に見たときの表示」です。その間にデータが変わっても、戻ったときに**古い控えが出る**ことがあります。

- 投稿を編集して一覧に戻ったら、編集前のタイトルが出ている
- カートに追加したのに、戻った画面では件数が前のまま

サーバー側のデータは新しくなっていても、ブラウザが持っている控えが古いと、ユーザーには古い画面が見えてしまいます。

## 再検証は、ブラウザの控えにも効く

サーバー側のキャッシュを捨てる `revalidatePath` は、**ブラウザの Router Cache にも「その控えは古い」と伝えます**。Server Action（サーバー側で動く関数）でデータを更新し、`revalidatePath` を呼ぶと、サーバーの保存とブラウザの控えがまとめて最新化されます。

```ts
// app/posts/actions.ts
"use server";

import { revalidatePath } from "next/cache";

export async function updatePost(formData: FormData) {
  await fetch("https://api.example.com/posts", {
    method: "POST",
    body: formData,
  });

  revalidatePath("/posts"); // サーバーの保存もブラウザの控えも最新化
}
```

さらに、フォームの送信などで Server Action を実行すると、Next.js は今表示しているルートの控えを自動で更新します。だから「更新ボタンを押したら、その場で新しい内容に変わる」という挙動になります。

## クライアントから手動で捨てる

データ更新を伴わず、ただ「今の画面を最新の状態で取り直したい」場合は、クライアントコンポーネントから `router.refresh()` を呼びます。

```tsx
"use client";

import { useRouter } from "next/navigation";

export function RefreshButton() {
  const router = useRouter();
  return (
    <button type="button" onClick={() => router.refresh()}>
      最新の状態に更新
    </button>
  );
}
```

`router.refresh()` は、今いるルートの控えを捨てて、サーバーから取り直します。入力中のフォームの状態などは保ったまま、表示するデータだけを新しくできます。

## なし・あり・再検証

| 状態 | 動き | 速さ | 鮮度 |
|------|------|------|------|
| 控えなし | 遷移のたびサーバーに取りに行く | 遅い | 常に最新 |
| 控えあり | ブラウザの控えで瞬時に遷移 | 速い | 控えた時点で固定 |
| 再検証 / refresh | 控えを捨てて取り直す | 速さは維持 | 操作後に最新へ |

ブラウザの控えで遷移を速くしておき、データを変えたときは `revalidatePath` や `router.refresh()` で控えを捨てて最新に戻す。これが基本の組み立てです。

なお、ここで扱ったのはブラウザ側の控えです。サーバー側の「取得したデータ」や「組み立てた HTML」の保存は、段階の違う別のキャッシュです。このレッスンではブラウザの控えに絞っています。

## まとめ

- Router Cache は訪れたページの表示をブラウザが控えておく仕組み
- 戻る・進むが瞬時になる代わりに、控えが古くなることがある
- `revalidatePath` はサーバーの保存もブラウザの控えもまとめて捨てる
- 更新を伴わず取り直したいときは `router.refresh()`
