# 色のアクセシビリティ — おしゃれな薄いグレーは誰を置き去りにするか

## 今日のゴール

- コントラスト比と「4.5 : 1」という基準の意味を知る
- 色の見え方は人によって違うこと（色覚多様性）を知る
- 「色だけで伝えない」という設計原則を知る

## 読めない「おしゃれ」

白い背景に薄いグレーの文字。余白たっぷりのモダンなデザインでよく見ますし、AI に UI を作らせても、補足テキストはたいてい薄いグレーになります。

```css
.note {
  color: #cccccc; /* 白背景に、上品な薄グレー */
  background: #ffffff;
}
```

制作した本人の環境では読めます。しかし、明るい屋外でスマホを見る人、年齢で水晶体が濁り始めた人、ロービジョン（弱視）の人にとって、この文字は**存在しないのと同じ**になり得ます。

「読める / 読めない」は主観の問題に見えて、実は**数値で判定できる領域**です。

## コントラスト比 — 読みやすさの物差し

文字色と背景色の明るさの差は、**コントラスト比**という数値で表せます。「白と白」が 1 : 1、「白と黒」が最大の 21 : 1 です。

Web のアクセシビリティ基準（WCAG）は、ここに具体的な合格ラインを引いています。

| 対象 | 合格ライン（AA） |
|------|----------------|
| 通常サイズの文字 | **4.5 : 1 以上** |
| 大きい文字（約 24px 以上など） | 3 : 1 以上 |
| アイコンや入力欄の枠線など | 3 : 1 以上 |

先ほどの薄グレー `#cccccc` ×白背景は約 **1.6 : 1**。基準の 3 分の 1 で、大幅な不合格です。`#767676` あたりでようやく 4.5 : 1 に届きます。「思ったより濃くしないと受からない」というのが、ほとんどの人の最初の感想です。

実物で見比べてみてください（背景はすべて白で固定しています）。

計算は道具に任せます。

- **ブラウザの開発者ツール**: 文字色の色見本をクリックするとコントラスト比と合否が表示される
- **WebAIM Contrast Checker** などの Web ツール: 2 色を入れるだけ

つまり「この色、読みにくくない？」という議論は、感想戦ではなく**数値の確認**で決着できます。AI に直させるときも「もっと濃く」ではなく「**コントラスト比 4.5 : 1 以上にして**」と言えば、一発で意図が伝わります。

## 色の見え方は、思っているより多様

もう 1 つの前提が、**色の見え方は人によって違う**ことです。

特定の色の組み合わせ（代表的には赤と緑）の区別がつきにくい**色覚多様性**は、日本では男性の約 5%（20 人に 1 人）、女性の約 0.2% にみられます。学校の 1 クラスに 1 人いる計算で、まったく珍しくありません。

このとき何が起きるかというと、たとえば:

- 「赤がエラー、緑が成功」のメッセージが、**同じような色**に見える
- グラフの折れ線が、色では区別できない

ダークモードも関係します。ライトモードでは合格だった色の組が、暗い背景に置いた途端に不合格になることは珍しくありません。**モードごとにコントラストは別物**として確認が必要です。

## 原則 — 色だけで伝えない

ここから導かれる設計原則が 1 つあります。

> **色は補助。情報そのものは、色以外の手段でも伝える。**

| ❌ 色だけ | ✅ 色 + 別の手段 |
|----------|----------------|
| エラーの入力欄を赤枠にする | 赤枠 + **エラーメッセージのテキスト** + アイコン |
| 必須項目を赤いアスタリスクだけで示す | 「必須」という**文字**のラベル |
| グラフの系列を色だけで分ける | 色 + **線種や形**（実線 / 破線、丸 / 四角）+ 直接ラベル |
| 「緑のボタンを押してください」という説明 | 「**［保存］ボタン**を押してください」と名前で呼ぶ |

```tsx
// ✅ 色（赤）はあくまで強調。情報はテキストとアイコンが運ぶ
<p role="alert" className="text-red-700">
  <span aria-hidden="true">⚠ </span>
  メールアドレスの形式が正しくありません
</p>
```

色覚の話だけではありません。色が見えない読み上げ環境、白黒印刷、強い日差しの画面。「色以外でも伝わるか」という問いは、**あらゆる劣化条件への保険**になります。

## AI の出力を見るポイント

色まわりで AI のコードを見るときは、この 3 つです。

1. **補足テキストの色が薄すぎないか**: グレー系の文字を見たら開発者ツールでコントラスト比を確認。4.5 : 1 が合格ライン
2. **状態を色だけで伝えていないか**: エラー・成功・選択中に、テキストやアイコンが添えてあるか
3. **ダークモードでも確認したか**: モードごとにコントラストは別物

そして余談をひとつ。コントラスト比を測って色を直す作業は、デザイナーの領分に踏み込むようでためらわれるかもしれません。しかし「4.5 : 1 を切っています」は感想ではなく測定結果です。**数字で話せるアクセシビリティは、職種を越えて通じる共通言語**になります。

## まとめ

- コントラスト比は読みやすさの物差し。通常文字は 4.5 : 1 以上（WCAG AA）
- 判定は感想でなく数値。開発者ツールで測れる
- 色覚多様性は男性の約 5%。色は補助、情報は色以外でも伝える
- ダークモードはコントラストを別物として再確認
