# Web パフォーマンス — Core Web Vitals で「速さ」を数値化する

## 今日のゴール

- Core Web Vitals という指標があることを知る
- LCP・CLS・INP がそれぞれ何を測っているかを知る
- Lighthouse で計測する方法を知る

## 「速い」を数値にする

「このサイト、なんか遅い」。感覚的にはわかるけれど、何がどう遅いのかを言葉にするのは難しいです。

Google はこの「速さ」を数値化するために **Core Web Vitals**（コアウェブバイタル）という 3 つの指標を定義しました。

## 3 つの指標

### LCP — 一番大きいコンテンツが表示されるまでの時間

**Largest Contentful Paint** の略です。ページを開いてから、画面上で一番大きい要素（メイン画像や見出しのテキストブロック）が表示されるまでの時間を測ります。

| 評価 | 基準 |
|------|------|
| 良好 | 2.5 秒以内 |
| 改善が必要 | 2.5 〜 4 秒 |
| 不良 | 4 秒超 |

ユーザーが「このページ、表示が速いな」と感じるかどうかに直結する指標です。

### CLS — レイアウトのズレ

**Cumulative Layout Shift** の略です。ページの読み込み中に要素の位置がガクッとずれる現象を数値化します。

記事を読んでいる途中に広告が表示されて、文章の位置がずれてしまった経験はないでしょうか。あの「ガクッ」を測っています。

| 評価 | 基準 |
|------|------|
| 良好 | 0.1 以下 |
| 改善が必要 | 0.1 〜 0.25 |
| 不良 | 0.25 超 |

画像に `width` と `height` を指定することで、表示される前からスペースを確保して CLS を防げます。

### INP — 操作に対する応答速度

**Interaction to Next Paint** の略です。ユーザーがクリックやキー入力をしてから、画面が更新されるまでの時間を測ります。

| 評価 | 基準 |
|------|------|
| 良好 | 200ms 以内 |
| 改善が必要 | 200 〜 500ms |
| 不良 | 500ms 超 |

ボタンを押してから反応するまでに 500ms かかると、「固まった？」と感じます。

## Lighthouse で計測する

Core Web Vitals はブラウザの DevTools に組み込まれた **Lighthouse** で計測できます。

1. DevTools を開く（右クリック → 検証）
2. 「Lighthouse」タブを開く
3. 「Analyze page load」をクリック

数十秒後に、4 つのカテゴリでスコアが表示されます。

| カテゴリ | 何を測るか |
|---------|-----------|
| Performance | 表示速度（LCP, CLS, INP を含む） |
| Accessibility | アクセシビリティ |
| Best Practices | セキュリティ、画像最適化など |
| SEO | 検索エンジン対応 |

各カテゴリ 0〜100 点で評価され、改善すべき項目が具体的に一覧されます。

## Next.js が自動でやっている最適化

Next.js を使っているプロジェクトでは、フレームワークがいくつかのパフォーマンス最適化を自動で行っています。

- **コード分割**: ページごとに JavaScript を分割し、必要な分だけ読み込む
- **画像最適化**（`next/image`）: 画像のリサイズ、WebP 変換、遅延読み込みを自動で行う
- **フォント最適化**（`next/font`）: Web フォントの読み込みを最適化し、CLS を防ぐ
- **Server Components**: JavaScript をブラウザに送らないことで、バンドルサイズを削減する

これらは Next.js を使うだけで有効になります。「パフォーマンスのために Next.js を選ぶ」という判断の裏には、こうした自動最適化があります。

## まとめ

- Core Web Vitals は「速さ」を数値化する 3 つの指標です
- **LCP**: 大きいコンテンツの表示時間（2.5 秒以内が良好）
- **CLS**: レイアウトのズレ（0.1 以下が良好）
- **INP**: 操作への応答速度（200ms 以内が良好）
- Lighthouse で計測でき、改善項目が具体的にわかります
- Next.js はコード分割、画像最適化、Server Components などを自動で行い、パフォーマンスを底上げしています
