# アロー関数とスコープ — `=>` の正体と変数が見える範囲

## 今日のゴール

- アロー関数が `function` の省略形であることを知る
- JavaScript の変数には「見える範囲（スコープ）」があることを知る
- `const` と `let` の使い分けの理由を知る

## `=>` は何者か

Next.js のコードでは、こういう書き方がよく出てきます。

```tsx
const handleClick = () => {
  console.log("クリックされました");
};
```

この `=>` を含む書き方を**アロー関数**と呼びます。実は、以下の `function` を使った書き方と同じことをしています。

```javascript
function handleClick() {
  console.log("クリックされました");
}
```

どちらも「処理をまとめて名前を付ける」ための**関数**です。アロー関数は `function` という単語を省略して `=>` で書く、より短い記法です。

### なぜアロー関数が多いのか

Next.js のコードでは `function` よりアロー関数が多く使われます。理由は 2 つあります。

1 つ目は**短く書ける**こと。特に引数が 1 つで処理が 1 行のとき、さらに省略できます。

```javascript
// 通常のアロー関数
const double = (n) => {
  return n * 2;
};

// 省略形: { } と return を省略できる
const double = (n) => n * 2;
```

2 つ目は **`this` の扱いが違う**こと。`function` で書いた関数は、呼び出し方によって `this` が変わります。アロー関数は、定義した場所の `this` をそのまま使います。React のコンポーネント内ではアロー関数のほうが予測しやすい動きをします。

`this` の詳細は今は覚えなくて大丈夫です。「`=>` は function の短縮形で、React ではこちらが一般的」と知っておけば十分です。

## スコープ — 変数が見える範囲

JavaScript には**スコープ**という仕組みがあります。変数は宣言した場所によって「見える範囲」が決まり、その外からはアクセスできません。

```javascript
function greet() {
  const message = "こんにちは";
  console.log(message);  // "こんにちは" ← 見える
}

greet();
console.log(message);  // エラー！ ← 関数の外からは見えない
```

`message` は `greet` 関数の中で宣言されたので、関数の外からは見えません。これを**関数スコープ**と呼びます。

### ブロックスコープ

`const` と `let` で宣言した変数は、`{ }` の中だけで有効です。これを**ブロックスコープ**と呼びます。

```javascript
if (true) {
  const x = 10;
  console.log(x);  // 10 ← 見える
}
console.log(x);  // エラー！ ← { } の外からは見えない
```

`{ }` が壁になって、中で宣言した変数が外に漏れません。

### なぜスコープが大切か

スコープがなかったら、すべての変数がプログラム全体から見えてしまいます。

```javascript
// もしスコープがなかったら...
const count = 0;       // カウンター用
const count = "3件";   // 表示用 → 名前が衝突！
```

CSS のグローバルスコープ問題と同じです。CSS はファイルを分けてもスタイルが全部混ざりました。JavaScript は `{ }` で壁を作ることで、同じ問題を言語レベルで防いでいます。

## `const` と `let` — なぜ 2 つあるのか

変数を宣言するキーワードは `const` と `let` の 2 つがあります。

```javascript
const name = "田中";    // 再代入できない
let count = 0;          // 再代入できる

name = "佐藤";  // エラー！ const は変えられない
count = 1;      // OK
```

| キーワード | 再代入 | 用途 |
|-----------|--------|------|
| `const` | できない | ほとんどの変数 |
| `let` | できる | カウンターなど変更が必要な場合 |

実務では**ほとんどの変数を `const` で書きます**。「この変数は途中で変わらない」とコードで宣言できるため、読む人にとって安心です。`let` は「この変数はどこかで変わる」というサインになるので、本当に必要なときだけ使います。

### var は使わない

古い JavaScript には `var` というキーワードもありました。`var` にはブロックスコープが効かないという問題があります。

```javascript
if (true) {
  var x = 10;
}
console.log(x);  // 10 ← var は { } の外に漏れる！
```

`const` / `let` はこの問題を解決するために追加されました。新しいコードでは `var` は使いません。

## まとめ

- `=>` はアロー関数。`function` の省略形で、React / Next.js のコードではこちらが一般的です
- JavaScript の変数には**スコープ**（見える範囲）があり、`{ }` の中で宣言した変数は外から見えません
- `const` は再代入できない変数、`let` は再代入できる変数。ほとんどの場面で `const` を使います
- `var` はスコープの問題があるため、現在は使いません
