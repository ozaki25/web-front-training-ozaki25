# HTTPS と TLS — 🔒 マークが保証していること

## 今日のゴール

- 🔒 が「暗号化」と「相手が本物か」の 2 つを保証すると知る
- なぜ http では盗聴・改ざん・なりすましが起きるか知る
- 証明書が「身元の証明」である仕組みを知る

## 🔒 は何を約束しているのか

アドレスバーの 🔒 は、その通信が TLS で保護された **HTTPS** であることを示します。TLS が保証するのは 2 つです。

- **暗号化**: 途中の誰かに中身を読まれない
- **身元の確認**: つないでいる相手が、本物の `example.com` である

http（TLS なし）だと、この 2 つが両方とも欠けます。

## http が危ない理由

通信は、あなたと相手の間にある多くの機器（Wi-Fi ルーター、プロバイダなど）を経由します。http では中身が平文なので、経路上の誰かが次のことをできてしまいます。

- **盗聴**: パスワードやカード番号をそのまま読む
- **改ざん**: 表示内容や返ってくるコードを書き換える
- **なりすまし**: 偽のサーバーが本物のふりをして応答する

公衆 Wi-Fi が危ないと言われるのは、この経路に他人が入り込みやすいからです。

## 暗号化だけでは足りない

中身を暗号化しても、その相手が本物かどうかが分からなければ、偽サーバーと安全に通信しているだけになってしまいます。

そこで **証明書**が使われます。信頼された発行機関（認証局）が「このドメインの持ち主は本物だ」と署名したものです。

ブラウザはこの署名を検証して、相手の身元を確かめます。🔒 は、この検証まで通ったことの印です。

## まとめ

- HTTPS（TLS）は暗号化と身元確認の 2 つを保証する
- http は平文なので、盗聴・改ざん・なりすましが起きる
- 証明書は認証局が署名した身元の証明で、🔒 はその検証済みの印
