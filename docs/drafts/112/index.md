# JavaScript の妙な挙動 — その場で予想して当てるクイズ

いつもは読んで学ぶ回ですが、今日は違います。コードの実行結果を**先に予想してから**答えを開く、クイズ形式の回です。

## 今日のゴール

- JavaScript の意外な挙動の裏に、型変換・数値・配列・スコープの仕組みがあると知る
- 「+」「==」「sort()」「var」など、身近な機能に潜む癖を知る
- 一見ありえない結果も、変換や比較のルールを追えば説明できると知る

## 遊び方

各問題のコードを見て、実行結果を予想してください。予想したら「答えと理由」を開いて確かめます。全問、実際に Node で実行して確認済みです。

## 型変換

### 出題 1: `[] + []`

```js
[] + []
```

::: details 答えと理由
`""`（空文字）になります。

`+` は両辺を文字列や数値に変換しようとします。配列を文字列に変換すると、要素をカンマでつないだ文字列になり、空の配列なら空文字です。だから `"" + ""` で空文字になります。
:::

### 出題 2: `[] + {}`

```js
[] + {}
```

::: details 答えと理由
`"[object Object]"` になります。

配列は空文字に変換されます。オブジェクトは既定で `"[object Object]"` という文字列に変換されます。空文字とこの文字列をつなげると `"[object Object]"` が残ります。
:::

### 出題 3: `true + true`

```js
true + true
```

::: details 答えと理由
`2` になります。

`+` が数値として計算しようとするとき、真偽値は `true` が `1`、`false` が `0` として扱われます。だから `true + true` は `1 + 1` と同じで `2` になります。
:::

### 出題 4: `"5" - 1` と `"5" + 1`

```js
"5" - 1
"5" + 1
```

::: details 答えと理由
上は `4`、下は `"51"` になります。

`-` は常に両辺を数値に変換してから計算します。`"5"` は数値の `5` になり、`5 - 1` で `4` です。一方 `+` は、どちらかが文字列なら文字列同士をつなげる動作を優先します。`"5"` はそのまま文字列として扱われ、`1` も文字列の `"1"` に変換されるので、`"5" + "1"`、つまり `"51"` になります。
:::

## 数値

### 出題 5: `typeof NaN`

```js
typeof NaN
```

::: details 答えと理由
`"number"` になります。

NaN は「数値ではない」という意味の値ですが、その正体は Number 型に属する特別な値です。だから `typeof` で確かめると `"number"` が返ります。
:::

### 出題 6: `NaN === NaN`

```js
NaN === NaN
```

::: details 答えと理由
`false` になります。

浮動小数点数の規格（IEEE 754）では、NaN は自分自身とも等しくないと決められています。JavaScript の数値はこの規格に従うので、`NaN === NaN` は常に `false` です。値が NaN かどうかを確かめたいときは `Number.isNaN()` を使います。
:::

### 出題 7: `0.1 + 0.2 === 0.3`

```js
0.1 + 0.2 === 0.3
```

::: details 答えと理由
`false` になります。

`0.1` や `0.2` は 2 進数の浮動小数点では正確に表せません。足すと `0.30000000000000004` という誤差入りの値になり、`0.3` と一致しないのです。金額計算でこの誤差が事故を起こすことがあります。
:::

## 配列とオブジェクト

### 出題 8: `[10, 1, 2].sort()`

```js
[10, 1, 2].sort()
```

::: details 答えと理由
`[1, 10, 2]` になります。数値の大きさ順ではありません。

`sort()` は既定では要素を**文字列**として比較します。`"10"` と `"2"` を文字列として比べると、先頭の `"1"` が `"2"` より小さい文字なので、`"10"` が `"2"` より前に来ます。数値の大きさで並べたいときは、比較関数 `(a, b) => a - b` を渡す必要があります。
:::

### 出題 9: `Array(3).map(x => 1)`

```js
Array(3).map(x => 1)
```

::: details 答えと理由
`[ <3 empty items> ]` になります。`[1, 1, 1]` にはなりません。

`Array(3)` は長さ 3 の配列を作りますが、中身は「空スロット」で値が入っていません。`map()` は値が入っているスロットにしか処理を適用しないので、空スロットはそのまま素通りします。値を入れたいときは `Array(3).fill(0)` のように明示的に埋める必要があります。
:::

### 出題 10: `Object.keys([1, 2, 3])`

```js
Object.keys([1, 2, 3])
```

::: details 答えと理由
`['0', '1', '2']` になります。数値ではなく文字列の配列です。

JavaScript の配列は、内部的には数値のインデックスをキーに持つオブジェクトです。`Object.keys()` はオブジェクトのキーを文字列で返すので、配列に使うと添字が文字列化された `'0'`、`'1'`、`'2'` が返ります。
:::

## 等価性とスコープ

### 出題 11: `null == undefined`

```js
null == undefined
```

::: details 答えと理由
`true` になります。

`==` の比較には特別ルールがあり、`null` と `undefined` はゆるく等しいと決められています。この性質を使って `if (value == null)` と書くと、「`null` でも `undefined` でもない」を一度に確かめられます。ただし数値の `0` とは別扱いで、`null == 0` は `false` です。
:::

### 出題 12: `var` とループの中の `setTimeout`

```js
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
```

::: details 答えと理由
`3` が 3 回出力されます。`0, 1, 2` ではありません。

`var` は関数（またはグローバル）スコープなので、ループの 3 回の反復すべてが**同じ `i`** を共有します。`setTimeout` のコールバックが実際に実行されるのはループが終わったあとなので、その時点の `i` の値である `3` が使われます。`let` で書けば、反復ごとに新しい `i` が作られるので `0, 1, 2` と出力されます。
:::

### 出題 13: `[] == ![]`

```js
[] == ![]
```

::: details 答えと理由
`true` になります。配列が自分自身の否定と等しいという、直感に反する結果です。

まず `![]` を計算します。空配列は truthy（真として扱われる値）なので、それを否定した `![]` は `false` になります。次に `[] == false` を比較します。`==` は両辺を数値に変換して比べようとし、`[]` は空文字を経て `0` に、`false` も `0` になるので、結局 `0 == 0` で `true` になります。
:::

## AI が書いたコードにも同じ規則が効く

AI が書いたコードも、この言語の上で動きます。変換の規則を知っていれば、出てきたコードの結果が予想と違ったときも、動じずに原因を追えます。

## まとめ

- 「+」は文字列化を、「==」は特別な比較規則を経て値を変換する
- `sort()` は既定で文字列として比較し、`var` はループの反復間で同じ変数を共有する
- NaN は数値の一種で、自分自身とも等しくない
- 浮動小数点の誤差は、言語を問わず起こる計算の前提
