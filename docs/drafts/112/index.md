# JavaScript の妙な挙動 — その場で予想して当てるクイズ

いつもは読んで学ぶ回ですが、今日は違います。コードの実行結果を**先に予想してから**答えを開く、クイズ形式の回です。

## 今日のゴール

- JavaScript の意外な挙動の裏に、見た目だけでは分からない仕組みがあると知る
- `sort()` の既定順序や `var` のスコープなど、実務でも踏みやすい地雷を知る
- NaN・浮動小数点・`==` の特別な比較規則を知る

## 遊び方

各問題のコードを見て、実行結果を予想してください。予想したら「答えと理由」を開いて確かめます。全問、実際に Node で実行して確認済みです。

## 出題 1: `[10, 1, 2].sort()`

```js
[10, 1, 2].sort()
```

::: details 答えと理由
`[1, 10, 2]` になります。数値の大きさ順ではありません。

`sort()` は既定では要素を**文字列**として比較します。`"10"` と `"2"` を文字列として比べると、先頭の `"1"` が `"2"` より小さい文字なので、`"10"` が `"2"` より前に来ます。数値の大きさで並べたいときは、比較関数 `(a, b) => a - b` を渡す必要があります。
:::

## 出題 2: `var` とループの中の `setTimeout`

```js
for (var i = 0; i < 3; i++) {
  setTimeout(() => console.log(i), 0);
}
```

::: details 答えと理由
`3` が 3 回出力されます。`0, 1, 2` ではありません。

`var` は関数（またはグローバル）スコープなので、ループの 3 回の反復すべてが**同じ `i`** を共有します。`setTimeout` のコールバックが実際に実行されるのはループが終わったあとなので、その時点の `i` の値である `3` が使われます。`let` で書けば、反復ごとに新しい `i` が作られるので `0, 1, 2` と出力されます。
:::

## 出題 3: `typeof NaN`

```js
typeof NaN
```

::: details 答えと理由
`"number"` になります。

NaN は「数値ではない」という意味の値ですが、その正体は Number 型に属する特別な値です。だから `typeof` で確かめると `"number"` が返ります。
:::

## 出題 4: `NaN === NaN`

```js
NaN === NaN
```

::: details 答えと理由
`false` になります。

浮動小数点数の規格（IEEE 754）では、NaN は自分自身とも等しくないと決められています。JavaScript の数値はこの規格に従うので、`NaN === NaN` は常に `false` です。値が NaN かどうかを確かめたいときは `Number.isNaN()` を使います。
:::

## 出題 5: `0.1 + 0.2 === 0.3`

```js
0.1 + 0.2 === 0.3
```

::: details 答えと理由
`false` になります。

`0.1` や `0.2` は 2 進数の浮動小数点では正確に表せません。足すと `0.30000000000000004` という誤差入りの値になり、`0.3` と一致しないのです。金額計算でこの誤差が事故を起こすことがあります。
:::

## 出題 6: `null == undefined`

```js
null == undefined
```

::: details 答えと理由
`true` になります。

`==` の比較には特別ルールがあり、`null` と `undefined` はゆるく等しいと決められています。この性質を使って `if (value == null)` と書くと、「`null` でも `undefined` でもない」を一度に確かめられます。ただし数値の `0` とは別扱いで、`null == 0` は `false` です。「null は 0 みたいなもの」という思い込みは外れます。
:::

## AI が書いたコードにも同じ規則が効く

AI が書いたコードも、この言語の上で動きます。変換の規則を知っていれば、出てきたコードの結果が予想と違ったときも、動じずに原因を追えます。

## まとめ

- `sort()` は既定で文字列として比較し、`var` はループの反復間で同じ変数を共有する
- NaN は数値の一種で、自分自身とも等しくない
- 浮動小数点の誤差は、言語を問わず起こる計算の前提
