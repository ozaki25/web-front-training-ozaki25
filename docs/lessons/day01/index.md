# Day 1: HTML の基本構造

## 今日のゴール

- HTML ファイルの基本構造（DOCTYPE、html、head、body）を理解する
- 見出し・段落・リストを使ってページを作れるようになる
- タグには「意味（セマンティクス）」があることを理解する
- なぜセマンティックな HTML が重要なのかを説明できる

## HTML とは

HTML（HyperText Markup Language）は、Web ページの**構造**を記述するための言語です。ブラウザは HTML を読み取り、画面に表示します。

ここで大事なのは、HTML は「見た目」ではなく「構造と意味」を記述するものだということです。「この文章は見出しである」「これはリストの項目である」といった**意味づけ（マークアップ）**を行うのが HTML の役割です。見た目の調整は、後の Day で学ぶ CSS が担当します。

## 基本構造

すべての HTML ファイルは以下の構造を持ちます。新しいフォルダを作り、以下のコードを `index.html` というファイル名で保存してみましょう。

```html
<!DOCTYPE html>
<html lang="ja">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>ページタイトル</title>
  </head>
  <body>
    <p>Hello, World!</p>
  </body>
</html>
```

保存したら、ファイルをブラウザで開いてみましょう。ダブルクリックするか、ブラウザにドラッグ&ドロップすれば表示されます。

各要素の役割を見ていきます。

| 要素 | 役割 |
|------|------|
| `<!DOCTYPE html>` | この文書が HTML5 であることをブラウザに宣言する。これがないとブラウザが古い互換モードで表示してしまうことがある |
| `<html lang="ja">` | HTML 文書のルート（最上位）要素。`lang="ja"` はページの言語が日本語であることを示す |
| `<head>` | ページのメタ情報（タイトル、文字コード、説明文など）を置く場所。画面には表示されない |
| `<body>` | 画面に表示される内容をすべて書く場所 |

### `<head>` の中身をもう少し詳しく

```html
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>ページタイトル</title>
</head>
```

- **`charset="UTF-8"`** — 文字コードの指定です。UTF-8 は日本語を含む世界中の文字を扱える標準的な文字コードです。これを書かないと日本語が文字化けする可能性があります
- **`viewport`** — スマートフォンでの表示を適切にするための設定です。これがないとスマホで見たときにページが極端に小さく表示されます
- **`<title>`** — ブラウザのタブに表示されるタイトルです。検索エンジンの検索結果にも使われます

### `lang` 属性はなぜ重要か

`<html lang="ja">` の `lang="ja"` は小さな記述ですが、重要な意味があります。

- **スクリーンリーダー**（視覚障害のある方が使う画面読み上げソフト）が正しい言語で読み上げるために使います。`lang="ja"` があれば日本語の発音で読み上げ、`lang="en"` なら英語の発音になります
- ブラウザの翻訳機能が、ページの言語を正しく判定するためにも使います

このように、HTML の属性には「機械やツールがページの情報を正しく理解する」ための手がかりとしての役割があります。こうした配慮を**アクセシビリティ**（誰でも使えるようにすること）と呼びます。

## 見出し — ページに構造を与える

`<h1>` が最も大きく、`<h6>` が最も小さい見出しです。

```html
<h1>大見出し</h1>
<h2>中見出し</h2>
<h3>小見出し</h3>
```

> **ポイント**: `<h1>` は 1 ページに 1 つが基本です。見出しレベルは飛ばさず順番に使いましょう（`<h1>` の次にいきなり `<h3>` を使わない）。

### なぜ見出しレベルの順番が大事なのか

見出しタグは「文字を大きくするためのタグ」ではありません。**ページの階層構造**を表すタグです。

本に例えると、`<h1>` は本のタイトル、`<h2>` は章のタイトル、`<h3>` は節のタイトルに相当します。スクリーンリーダーを使っている人は、見出しの一覧を使ってページ内を移動します。見出しレベルが飛んでいると、構造が壊れてしまい、ナビゲーションが困難になります。

文字の大きさを変えたいだけなら、後で学ぶ CSS を使います。

## 段落

```html
<p>これは段落です。文章のまとまりを表します。</p>
<p>別の段落は別の p タグで囲みます。</p>
```

改行したいだけで `<p>` を使うのではなく、「意味のひとまとまりの文章」を `<p>` で囲みます。

## リスト

**順序なしリスト**（箇条書き）— 項目の順番に意味がない場合:

```html
<ul>
  <li>りんご</li>
  <li>みかん</li>
  <li>バナナ</li>
</ul>
```

**順序ありリスト**（番号付き）— 手順など、順番に意味がある場合:

```html
<ol>
  <li>まず卵を割る</li>
  <li>かき混ぜる</li>
  <li>フライパンで焼く</li>
</ol>
```

`<ul>` と `<ol>` の使い分けも見た目ではなく意味で決めます。「順番を入れ替えても意味が変わらないか？」を基準にしましょう。買い物リストは `<ul>`、料理の手順は `<ol>` です。

## セマンティック HTML — タグに意味がある

HTML のタグにはそれぞれ意味があります。これを**セマンティクス（意味論）**と呼びます。

たとえば、以下の 2 つは見た目が似ていても意味がまったく違います。

```html
<!-- ❌ 見た目だけを変えている。意味がない -->
<span style="font-size: 24px; font-weight: bold;">お知らせ</span>

<!-- ✅ 「これは見出しである」という意味を持つ -->
<h2>お知らせ</h2>
```

なぜセマンティックなタグを使うべきなのでしょうか？

1. **アクセシビリティ**: スクリーンリーダーはタグの意味を読み取って「見出し」「リスト」「ナビゲーション」などをユーザーに伝えます。意味のないタグばかりだと、ページの構造が伝わりません
2. **SEO（検索エンジン最適化）**: 検索エンジンもタグの意味を読み取ってページの内容を理解します
3. **メンテナンス性**: コードを読む人（将来の自分を含む）が構造を把握しやすくなります

## 実際に書いてみよう

ここまで学んだタグを組み合わせて、自己紹介ページを作ってみましょう。先ほどの `index.html` を以下の内容に書き換えてください。

```html
<!DOCTYPE html>
<html lang="ja">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>自己紹介 - 山田太郎</title>
  </head>
  <body>
    <header>
      <h1>山田太郎</h1>
      <p>東京都在住のエンジニアです。</p>
    </header>

    <main>
      <section>
        <h2>自己紹介</h2>
        <p>Web 開発に興味があり、日々勉強しています。</p>
        <p>このサイトは HTML の学習のために作りました。</p>
      </section>

      <section>
        <h2>趣味</h2>
        <ul>
          <li>読書</li>
          <li>ランニング</li>
          <li>プログラミング</li>
        </ul>
      </section>

      <section>
        <h2>学びたい技術</h2>
        <ol>
          <li>HTML と CSS の基礎</li>
          <li>JavaScript</li>
          <li>React と Next.js</li>
        </ol>
      </section>
    </main>

    <footer>
      <p>© 2026 山田太郎</p>
    </footer>
  </body>
</html>
```

ブラウザで開くと、見た目はシンプルですが、構造としてはしっかりした HTML になっています。ここで使った新しいタグを確認しましょう。

| タグ | 意味 |
|------|------|
| `<header>` | ページやセクションのヘッダー（導入部分） |
| `<main>` | ページのメインコンテンツ。1 ページに 1 つだけ |
| `<section>` | 意味的にまとまったセクション |
| `<footer>` | ページやセクションのフッター |

これらは `<div>`（意味を持たない汎用コンテナ）でも代用できますが、セマンティックなタグを使うことでページの構造が明確になります。スクリーンリーダーは `<main>` を見つけて「ここからメインコンテンツです」とユーザーに伝えることができます。

名前や趣味の部分を自分の情報に書き換えて、ブラウザを更新してみてください。

## まとめ

- HTML は `<!DOCTYPE html>` → `<html>` → `<head>` + `<body>` という構造を持つ
- `<head>` にはメタ情報（文字コード、ビューポート、タイトル）を書く
- `<h1>`〜`<h6>` で見出し、`<p>` で段落、`<ul>`/`<ol>` + `<li>` でリスト
- タグは「見た目」ではなく「意味」で選ぶ。これがセマンティック HTML の基本
- `<header>`、`<main>`、`<section>`、`<footer>` などのセマンティックタグを使うと、ページの構造が明確になる
- `lang` 属性や見出しレベルの順守は、アクセシビリティの基本
