| テーマ | レッスンの中身 |

# 新しい鉱脈（4視点ブレストより）

シニアFE・トラブル対応・AI協働・設計判断の4視点で発想。複数視点で重複したものは確度が高い。

## 言語・計算の足元の罠（AIも地味に間違える）

| テーマ | レッスンの中身 | 確度 |
|-------|-------------|------|
| 0.1 + 0.2 ≠ 0.3 | なぜ計算がズレるか（2進数浮動小数点）。金額計算で1円ズレる事故。整数で持つ定石 | 視点1,2 |
| 日付とタイムゾーン | `new Date()` がどこの時刻か。サーバー(UTC)とブラウザ(JST)のズレ。1日ずれる現象 | 視点1,2 |
| 文字コードと文字化け | UTF-8 とは。`"𩸽".length` がズレる（サロゲートペア）。文字数制限で踏む | 視点1,2 |
| 「ない」の種類 | null / undefined / 空文字 / 0 / 空配列。`if(count)` で 0 が消える。`??` と `\|\|` の違い | 視点1,2 |
| JSON とシリアライズ | メモリ上のオブジェクトと送れる文字列の違い。Date や関数が消える理由。Server→Client で渡せる値の制限 | 視点1,2 |
| 文字列ソートの罠 | `[1,2,10].sort()` が `[1,10,2]` になる理由。日本語の五十音順が効かない | 視点2 |

## 実務で必ずハマる現象（ハマるまで存在を知らない）

| テーマ | レッスンの中身 | 確度 |
|-------|-------------|------|
| 競合状態（race condition） | 検索を速く打つと前の結果が後から上書きされる。投げた順と返る順が違う。AbortController | 視点2 |
| 本番だけ動かない | 開発と本番のビルドの違い。大文字小文字、環境変数の埋め込み、ミニファイ | 視点1,2 |
| 保存したのに変わらない | ブラウザ/ビルド/Service Worker のキャッシュ多層。ハードリロードで直る理由 | 視点1,2 |
| z-index が効かない | `z-index:9999` でも前に出ない。スタッキングコンテキストという見えない仕組み | 視点1 |
| 100vh がスマホで見切れる | アドレスバー込みで計算される問題。dvh/svh/lvh の登場 | 視点2 |
| メモリリーク | タイマーやリスナーが消えずに残る。なぜタブを開きっぱなしで重くなるか。useEffect の return | 視点2 |
| イベントの伝わり方 | クリックが DOM をどう伝播するか（バブリング）。モーダル外クリックで閉じる仕組み | 視点1 |

## AI と働く（自分からは絶対気づかない）

| テーマ | レッスンの中身 | 確度 |
|-------|-------------|------|
| AI の知識には賞味期限がある | 学習データのカットオフ。だから非推奨 API や古い書き方を出す。最新は自分で確認 | 視点3 |
| なぜ存在しない API を自信満々に作るか | ハルシネーション。AI は「正しい」でなく「それっぽく続く」を生成。もっともらしさ ≠ 正しさ | 視点3 |
| AI が「さっき言ったこと」を忘れる | トークンと文脈長。長い会話で古い指示が押し出される。要点の再提示 | 視点3 |
| AI は「あなたに同意したい」 | 迎合バイアス。「良いですか?」と聞くと「良いです」と返る。「3つ問題点を挙げて」と聞く | 視点3 |
| AI は最短で動く道を選ぶ | 「テスト通して」でテストを骨抜きにする。言葉通りに最適化する。指示の穴 | 視点3 |
| 動くコードと読めるコード | AI は動くコードを量産するが保守性は別。理解せずマージは時限爆弾 | 視点3,4 |
| エラーは AI への手土産 | 要約せず全文を貼る。スタックトレースの読み方。何を渡せば的確に直るか | 視点2,3 |

## 設計・判断（会議で置いていかれない）

| テーマ | レッスンの中身 | 確度 |
|-------|-------------|------|
| なぜ密結合は悪いのか | 1つ直すと10箇所壊れる正体。疎結合 = 相手の中身を知らなくても使える。影響範囲が読める | 視点4 |
| 技術的負債はなぜ返せないか | 妥協したコードは利子付きの借金。返済時間は誰も確保しない。意図的に借りて記録する | 視点4 |
| 動いたと正しいは別 | 偶然動いているコードは環境が変わると壊れる。「なぜ動くか」を説明できるか | 視点2,4 |
| 抽象化のしすぎ問題 | 「将来使うかも」の共通化が負債になる。Rule of Three（3回繰り返してから） | 視点4 |
| 可逆な決定と不可逆な決定 | やり直せる決定は速く、DB設計や公開APIは慎重に。悩む重さを可逆性で測る | 視点4 |
| エッジケースを考える習慣 | 空・0件・最大・異常入力で何が起きるか。「0件のときどうなる?」と問える視点 | 視点1,4 |
| 見積もりはなぜ当たらないか | 分かっている作業より調査・連携・手戻りが時間を食う。見積もりは幅であって約束ではない | 視点4 |

# さらに広げた鉱脈（9視点ブレストより）

UXデザイナー・QA・セキュリティ・SRE/インフラ・元初心者・アクセシビリティ・パフォーマンス・PM・CS基礎の9つの立場で発散。発想を絞らず全部残す。**複数の視点から独立に出てきたテーマは「ここで聞かなければ知らない × 実務で効く」の2軸を満たす確度が特に高い**ので先頭にまとめる。

## 横断で繰り返し出たテーマ（確度が高い）

| テーマ | レッスンの中身 | 出てきた視点 |
|-------|-------------|------------|
| デプロイ後の世界 / 環境とは | 自分のコードがローカルを離れ、ビルドされ、サーバーと他人のブラウザで動く。ローカル/ステージング/本番の違い。すべてのインフラ話の土台 | SRE・元初心者 |
| CORS と同一オリジンポリシー | 別オリジンの API を叩くとなぜ弾かれるか。ブラウザが「勝手に他サイトを読ませない」防御をしている。`*` 許可で穴を開ける罠 | 元初心者・セキュリティ |
| 認証と認可・トークンの保管場所 | ログイン状態が保たれる仕組み。Cookie/セッション/JWT。localStorage に入れると XSS で盗まれる、HttpOnly Cookie なら読めない | 元初心者・セキュリティ・SRE |
| 入力を信用しない（信頼境界） | クライアントのチェックは UX、本物の防御はサーバー。`maxlength` も `disabled` も外せる。XSS も SQLi もここから | セキュリティ・QA・元初心者 |
| N+1 / 直列ウォーターフォール | ループの中で fetch すると 100 件で 101 回通信。`await` を並べて直列に待つ。`Promise.all` でまとめる発想 | パフォーマンス・(バックエンド) |
| メモリリーク | 解除し忘れた listener / interval / `useEffect` の return。タブを開きっぱなしで重くなる正体。GC が回収できない参照 | パフォーマンス・CS基礎 |
| スタックトレースの読み方 | 赤い文字は「関数が呼ばれた履歴」。一番下／自分のコードの行から読む。AI に貼るとき何を渡すか | 元初心者・CS基礎・QA |
| ビルドとバンドル | 書いたコードはそのまま動いていない。TS/JSX が変換され複数ファイルが 1 つにまとまる。`import` 一行が数百 KB 増やす | 元初心者・パフォーマンス |
| 4 状態（ローディング/成功/失敗/空）とステートマシン | 非同期 UI には必ず 4 つの顔がある。AI は成功状態だけ作りがち。boolean を並べると矛盾した画面が出る | QA・CS基礎 |
| 計測してから改善 | 「速くなった気がする」「ここが遅そう」は当たらない。Lighthouse / A/B テストで数字で判断してから手を入れる | パフォーマンス・PM |
| 色だけで伝えない / タッチターゲット | 赤字だけのエラーは色覚特性で消える。小さすぎるボタンは誤タップを生む。色＋アイコン＋テキスト、最小 44px | アクセシビリティ・UXデザイナー |
| 正常系・準正常系・異常系 / エラーは仕様 | 「うまくいくケース」しか頭にないのが初心者。例外の見せ方は信頼・離脱・問い合わせコストに直結する事業判断 | QA・PM |
| 質問・再現手順の型 | 「動きません」では誰も助けられない。再現手順・期待・実際・試したこと。AI への指示精度にも直結 | 元初心者・QA |

## UX デザイナー視点（なぜこの見た目・動きが良いか）

| テーマ | レッスンの中身 |
|-------|-------------|
| タッチターゲットと最小サイズ | 見た目の大きさと当たり判定は別物。なぜ 44px・48dp 推奨か。AI の小さいアイコンボタンの違和感 |
| タイポグラフィ三原則 | line-height・行長（日本語35〜45字）・字間。「なんか読みにくい」の正体 |
| ゲシュタルト原則 | 近接・類似・囲み。なぜ近いものは仲間に見えるか。余白とグルーピングが説明なしで構造を伝える |
| アフォーダンスとシグニファイア | 押せそう・入力できそうに見える手がかり。フラットデザインで操作性が下がる理由 |
| 良いエラー文の設計 | 「エラーが発生しました」がダメな理由。何が・なぜ・どうすればの3点、責めない言葉 |
| 体感時間とプログレス表現 | 同じ待ち時間でも進捗バーで短く感じる。「何も起きない」が最悪な理由 |
| ヒックの法則 | 選択肢が増えると決定時間が対数的に伸びる。CTA を絞る、段階的開示 |
| 認知負荷とワーキングメモリ | 一度に保持できる情報量の限界。フォームを分割しチャンク化する理由 |
| 視覚的階層 | すべて強調すると何も強調されない。サイズ・太さ・色・余白で「見てほしい順」を作る。全部太字の AI 成果物 |
| デフォルトの力 | 人は初期値を変えない。初期状態が行動を左右する。ダークパターンとの境界 |
| 破壊的操作の設計 | 削除を赤く、確認より Undo が優れる場面。誤操作前提で「戻れる」を設計 |
| エンプティステート | データ0件の初回画面を放置すると「壊れてる?」。空状態を次の一歩に変える |
| 視線の流れ（F字・Z字） | 人は読まず拾い読み。重要要素を視線の物理的な流れに沿って置く |
| マイクロインタラクションと4状態 | hover/active/focus/disabled をなぜ全部用意するか。「押した感」が信頼を生む |
| 一貫性と予測可能性 | 同じ機能のボタンが画面ごとに違うと学習コストが増える。考えなくても使える |
| 割り込み UI のコスト | モーダル/トースト/インラインの使い分け。作業を止める「割り込み」の重さ |
| フォントが運ぶ印象 | ゴシック=無機質、明朝=格調。可読性とは別軸の「声色」。デザイナーの「堅すぎる」が分かる |
| フォーム入力体験 | ラベルの位置（プレースホルダ単独がダメな理由）、入力例、適切なキーボード表示 |
| デザイナーと噛み合う言葉 | ジャンプ率・あしらい・トンマナ、Figma のオートレイアウト/バリアントが CSS の何に当たるか |

## QA 視点（品質をどう捉えるか）

| テーマ | レッスンの中身 |
|-------|-------------|
| 「動いた」と「正しい」は違う | 表示された/正しい値/壊れない、は別物。確認の解像度を段階で言語化 |
| 正常系・準正常系・異常系 | 入力ミスやキャンセル（準正常）、通信断やサーバエラー（異常）の3層。AI に「異常系も考慮して」と言える |
| なぜ仕様書通りでバグるか | 仕様には必ず「書かれていないこと」がある。バグは実装ミスより仕様の空白に落ちる |
| 境界値 | 「100円以上で無料」なら99/100/101が危ない。`>` か `>=` か。0・1・最大・空が壊れやすい |
| 再現手順が9割 | 「たまにおかしい」では直せない。いつ・どこで・何をしたら・何が起きて・何を期待したか |
| デグレと回帰 | 1箇所直すと別が壊れる正体（共有状態・共通関数・副作用）。「ここを直すと何に影響?」 |
| 再現性のないバグ | 押す順番・ダブルクリック・通信の速遅。状態とタイミングという見えない変数 |
| ハッピーパス症候群 | 作り手は正しい操作しか試さない（確証バイアス）。第三者視点・破壊的に触る視点 |
| 壊し方のカタログ | 空欄・超長文・絵文字・タグ・マイナス・全角・連打。フォームを壊す定番パターン一覧 |
| 状態の組み合わせ爆発 | ログイン×データ有無×権限×端末。全部は無理、重要度×頻度×リスクで選ぶ |
| データ件数で壊れる | 0件・1件・大量。空状態の表示崩れ、「該当なし」の出し方 |
| 非同期UIの4状態 | ローディング/成功/失敗/空。画面の必須セットとして捉える |
| バグの深刻度と優先度は別 | 見た目のズレと決済失敗は重さが違う。限られた時間でどれから直すか |
| 「直りました」を信じない | 報告バグが直ったか＋周辺を壊していないか。AI の「修正しました」を鵜呑みにしない |
| テストデータの作り方 | 「田中太郎」だけでテストしない。1文字の名前、記号入り住所、桁あふれ |
| 二重送信・連打・戻るボタン | ユーザは想定外の操作をする前提。送信連打で二重注文、ブラウザバックで壊れる |
| 開発者コンソールは検査道具 | 動いて見えても裏で赤いエラー、Network の失敗リクエスト。特別なツールなしの第一歩 |
| 受け入れ基準 | 「完成」を誰が決めるか。Acceptance Criteria を先に渡すと手戻りが減る |

## セキュリティ視点（動くけど危険）

| テーマ | レッスンの中身 |
|-------|-------------|
| 信頼境界 | クライアントのチェックは UX、サーバーが本物の防御。リクエストは curl で直接送れる |
| SQL インジェクション | API に送る検索ワードが SQL に連結されると全件抜かれる。文字列連結 vs プレースホルダ |
| オープンリダイレクト | `?redirect=` を弾かないとフィッシングの踏み台。ホワイトリスト/相対パス限定 |
| クリックジャッキング | 透明な iframe を重ねてボタンを押させる。`X-Frame-Options`/`frame-ancestors` |
| トークンの保管場所 | localStorage は XSS で盗まれる、HttpOnly Cookie は JS から読めない。場所の選択が防御 |
| なぜ HTTPS か | カフェの Wi-Fi で平文は覗ける。盗聴・改ざん・なりすまし。Mixed Content |
| 最小権限の原則 | 読み取りで足りるのに admin キー。漏れたときの被害は権限の広さで決まる（blast radius） |
| 同一オリジンと CORS | ブラウザの基本防御線。`*` でとりあえず許可して穴を開ける初心者を SOP の目的から正す |
| ログに秘密を書かない | `console.log(user)` でトークン流出、Sentry にボディごと送る事故。ログは第三者が見る |
| レート制限とブルートフォース | 回数制限がないとパスワード総当たり。本物の制限はサーバー側 |
| IDOR | `/orders/123` を `124` に変えたら他人の注文。認証と認可は別。id を知る≠アクセス権 |
| ファイルアップロードの危険 | `.jpg` でも中身は実行可能かも。拡張子だけでなく MIME・サイズ・保存場所 |
| エラーの出しすぎ | スタックトレースや「このメールは未登録」が内部構造やアカウント存在を漏らす |
| 第三者スクリプトの危険 | 広告・解析タグは全 DOM・Cookie・入力にアクセスできる。Magecart 型のカード盗用 |
| JWT は誰でも読める | Base64 デコードで丸見え。署名（改ざん検知）と暗号化（秘匿）は別。秘密を入れる事故 |
| パスワードは保存しない | 平文保存は漏洩時に全滅。ハッシュとソルト。AI にログイン機能を頼むときの判断軸 |
| CSP | XSS は防ぎきれない前提で、注入コードを実行させない最後の壁。`unsafe-inline` で台無し |
| ロックファイルとインストール時実行 | `npm install` 一発で数百パッケージが動く。`postinstall` で任意コード |

## SRE / インフラ視点（デプロイされた後の世界）

| テーマ | レッスンの中身 |
|-------|-------------|
| デプロイ後の世界 | ローカルから「他人のブラウザ＋どこかのサーバー」へ場が移る一枚絵。全インフラ話の土台 |
| CDN とは | 世界中にコピーを配るから近くから届く。オリジンとエッジキャッシュ、なぜ静的だけ乗りやすいか |
| ドメインと DNS | URL からサーバーに辿り着くまで。A/CNAME/名前解決、なぜ反映に時間がかかるか（TTL） |
| HTTPS と TLS 証明書 | 鍵マークの裏。証明書は何の保証か、なぜ期限切れで真っ赤になるか |
| なぜサーバーは複数あるか | スケールアップ vs スケールアウト。増やせる前提のステートレス設計 |
| ロードバランサ | 複数サーバーへの交通整理。1台落ちても回す（ヘルスチェック）、入口は1つに見える |
| なぜ 503/502 が出るか | 404/500/502/503/504 の違い。サーバー・LB・タイムアウトのどこで起きたか |
| ロールバック | 原因究明より先に直前の正常状態へ戻す。戻せないもの（DB変更）という落とし穴 |
| Blue/Green とカナリア | 別の場所に立てて一気に切替、問題なら即戻す。一部ユーザーに先出し |
| コールドスタート | 眠っている関数を起こす遅延。サーバーレス/エッジで起きる、ウォーム対策 |
| エッジとは | ユーザーのすぐ隣でコードを動かす。CDN が静的配信ならエッジは処理の近接化 |
| リージョンとレイテンシ | サーバーの地理的位置が速度を決める。光速という壁、DB の近くに置く理由 |
| キャッシュと TTL | 速さと鮮度のトレードオフ。「デプロイしたのに反映されない」、パージ/無効化 |
| ヘルスチェックと死活監視 | `/health` は何のためか。liveness と readiness、落ちたサーバーの自動切り離し |
| タイムアウトとリトライ | 待ち続けない・諦めない。無限リトライが雪だるま式障害を起こす |
| レート制限 | 429 の意味。フロントが大量に叩くと自分の首を絞める |
| ステートレスとセッション | サーバーが増えてもログイン状態を保てる理由。情報をどこに置くか、JWT との相性 |
| なぜ本番は壊れるか | 負荷スパイク・依存ダウン・設定ミス・容量超過。単一障害点（SPOF） |
| デプロイのライフサイクル | push→ビルド→デプロイ→トラフィック切替。PaaS がどこを肩代わりするか |

## PM / ビジネス視点（なぜ作るのか・何が事業価値か）

| テーマ | レッスンの中身 |
|-------|-------------|
