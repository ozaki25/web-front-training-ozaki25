# はじめに

## このコンテンツの目的

あなたはこれから Next.js を使ったプロジェクトに参加します。このコンテンツは、そのために必要な Web フロントエンドの知識を基礎から身につけるためのものです。

ゴールは**コードが書ける**ことではありません。AI ツールを使えば動くコードは作れます。このコンテンツで目指すのは、**フレームワークやライブラリが何を解決しているのかを理解し、設計やアーキテクチャの議論に参加できる**ようになることです。

## なぜ基礎から学ぶのか

プロジェクトでは Next.js、React、TypeScript、Tailwind CSS といった技術を使います。これらはとても便利ですが、その便利さは素の HTML/CSS/JavaScript が抱える課題を解決することで成り立っています。

例えば:

- React のコンポーネントが便利なのは、素の JavaScript で画面を書き換える DOM 操作が煩雑だから
- Tailwind CSS や CSS Modules が使われるのは、素の CSS にはスタイルのスコープという概念がなく、意図しない上書きが起きるから
- TypeScript が必要なのは、JavaScript だけでは実行するまでバグに気づけないから

基礎を飛ばしてフレームワークを学ぶと、「なんとなく動くが、なぜそう書くかわからない」状態になります。それでは設計の議論に参加できませんし、問題が起きたときに自分で判断できません。

このコンテンツでは、まず素の技術で「困る体験」をしてもらい、そのうえでフレームワークがどう解決しているかを学びます。

## 進め方

- **1日1レッスン、15分程度**で取り組めるボリュームです
- **順番に進めてください**。前のレッスンの知識を前提にしています
- 解説を読みながら、コード例をマネして実際に動かしてみてください
- 約2ヶ月（平日40日分）で完了する想定です

## 全体の流れ

1. **HTML/CSS** — Web ページの構造とスタイリングの基本を理解する
2. **JavaScript** — ブラウザ上での動的な処理と DOM 操作を学ぶ
3. **TypeScript** — 型による安全なコードの書き方を知る
4. **React** — コンポーネントベースの UI 構築と、素の JavaScript との違いを実感する
5. **Next.js** — App Router、Server Components など、React の上に何が加わるのかを理解する

各ステップは「前のステップで感じた課題を、次の技術がどう解決するか」という流れで繋がっています。
